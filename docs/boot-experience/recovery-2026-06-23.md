# Recovery Report - 2026-06-23

## Incident

After Plymouth customization tests, the system booted through:

* NeoVolt GRUB Theme
* LUKS Unlock Screen

However, all user logins failed.

The following message appeared:

```text
System is booting up. Unprivileged users are not permitted to log in yet.
```

## Symptoms

Observed failures:

* LightDM login blocked
* Autologin blocked
* TTY login blocked for normal users

## Investigation

The following components were verified:

* LightDM
* systemd-logind
* polkit
* accounts-daemon
* Cinnamon Desktop

All services were operational.

## Root Cause

The file:

```text
/run/nologin
```

remained present after boot.

This prevented authentication for non-privileged users.

## Recovery Procedure

Booted into Recovery Mode.

Executed:

```bash
rm -f /run/nologin
systemctl daemon-reexec
reboot
```

## Validation

Verified:

* systemctl --failed = 0
* LightDM active
* Cinnamon desktop loading normally
* Dual Boot operational
* LUKS operational
* NeoVolt GRUB operational

## Result

System fully recovered.

## Status

Resolved.

NeoVolt Linux Experience operational and stable.
